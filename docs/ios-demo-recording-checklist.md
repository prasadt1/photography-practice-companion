# Iris iPhone demo — recording checklist

Use with [`ios-demo-video-script.md`](ios-demo-video-script.md). ~10 minutes prep.

## Device

- [ ] iPhone plugged in, unlocked, **Developer Mode** on
- [ ] Xcode: **Iris** scheme → **your iPhone** (not Simulator)
- [ ] **Signing → Team** set (not None)
- [ ] Run (⌘R) — app opens on phone
- [ ] **Do Not Disturb** on; brightness ~70%

## App state

- [ ] **Continue in demo mode** → pick persona (or sign in on device if tested)
- [ ] Optional: Settings → Developer → paste Firebase uid for real portfolio
- [ ] Confirm **API OK** banner on launch (or ignore if offline test only)
- [ ] If Home is empty: one **Shoot → Gallery** → wait for critique

## Recording

- [ ] Control Center → **Screen Recording** → start
- [ ] Follow script table (~75s); speak slowly
- [ ] End on Home or springboard with Iris icon visible
- [ ] Stop recording → Photos app → trim start/end

## Optional B-roll (Mac)

- [ ] Safari: https://practice-companion-hackathon.web.app → My Work
- [ ] Quick cut in iMovie/CapCut between phone + web

## Devpost upload

- [ ] Video ≤ 3 min (script is ~75s)
- [ ] Link live web app in project description
- [ ] Mention iOS as field companion (Xcode install, not App Store required)

## If something breaks mid-record

| Issue | Fix |
|-------|-----|
| Analyze spinner stuck | Retry; keep app foreground; use Gallery on simulator |
| Mentor slow | Normal 10–90s; use suggested starter, wait |
| Empty Home | Shoot one photo first |
| Offline banner | Wi‑Fi/cellular on |
